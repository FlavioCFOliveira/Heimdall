# Heimdall v1.0.0 — Release Notes

**Release date:** 2026-04-27  
**Type:** General Availability (stable)  
**Tag:** `v1.0.0`

Heimdall 1.0.0 is the first stable, production-ready release of the Heimdall
DNS server.  All APIs, configuration formats, and wire behaviours are stable
and subject to the SemVer compatibility commitment below.

---

## Minimum Supported Rust Version (MSRV)

**Rust 1.94.0** — required for Rust Edition 2024 features and `let`-chain
stabilisation (RFC 2497).  The workspace is pinned to the nightly channel via
`rust-toolchain.toml` until Rust 1.94 is released as stable.

MSRV increases will be treated as breaking changes and will require a major
version bump.

---

## SemVer Stability Commitment (ENG-162..164)

From v1.0.0, Heimdall follows [Semantic Versioning 2.0.0](https://semver.org):

| Surface | Stability guarantee |
|---|---|
| TOML configuration schema | No breaking changes in 1.x |
| Admin RPC (JSON/UDS) command set | No breaking changes in 1.x |
| Prometheus metrics names + label set | No breaking changes in 1.x |
| `heimdall-core` public Rust API | No breaking changes in 1.x |
| `heimdall-roles` public Rust API | No breaking changes in 1.x |
| `heimdall-runtime` public Rust API | No breaking changes in 1.x |
| DNS wire behaviour (RFC conformance) | No regressions in 1.x |
| CLI flags and exit codes | No breaking changes in 1.x |

### Deprecation policy

A feature may be deprecated in a minor release (1.x.0) with a deprecation
warning in the documentation and release notes.  It is removed no earlier than
the next major version (2.0.0), and no sooner than 6 months after deprecation.

### MSRV policy

MSRV increases are announced in the minor release that introduces them, at
least one release in advance.  An MSRV increase is a breaking change requiring
a major version bump.

## Long-Term Support (LTS)

The `v1.0` branch is the LTS branch:

- **Support window:** 12 months from GA (2026-04-27 — 2027-04-27).
- **Backports:** security fixes and critical bug fixes only.
- **Releases:** patch versions (v1.0.1, v1.0.2, …) as needed.
- **Downstream migration:** guidance published 3 months before end-of-life.

See `docs/adr/<N>-lts-model.md` for the full LTS policy.

## Upgrade Path

### From v1.0.0-rc.1

No changes required.  Drop-in upgrade:

```sh
cosign verify-blob \
  --bundle heimdall-v1.0.0-x86_64-unknown-linux-musl.cosign.bundle \
  heimdall-v1.0.0-x86_64-unknown-linux-musl

systemctl stop heimdall
install -m 755 heimdall-v1.0.0-x86_64-unknown-linux-musl /usr/sbin/heimdall
systemctl start heimdall
```

### From v0.9.0-alpha.1 or v0.9.0-beta.1

No configuration migration required.  Follow the same drop-in upgrade process.
The cache, NTA store, and zone data are forward-compatible across the entire
pre-release → 1.0.0 range.

## Key Features

- **DNSSEC**: Full RFC 4033–4035, aggressive NSEC/NSEC3 (RFC 8198), NSEC3
  iteration cap (RFC 9276), KeyTrap mitigations (RFC 9364), RFC 5011 trust
  anchor management, NTA store (RFC 7646).
- **QNAME minimisation**: Strict and relaxed modes (RFC 9156).
- **0x20 case randomisation**: Anti-spoofing per draft-vixie-dnsext-dns0x20.
- **Response Policy Zones (RPZ)**: QNAME/CIDR/NSDNAME, multi-zone
  first-match-wins, AD suppression + EDE, dynamic reload.
- **Admin RPC**: JSON over Unix domain socket.
- **HTTP observability**: `/healthz`, `/readyz`, `/metrics`, `/version`.
- **Runtime hardening**: seccomp-bpf, privilege drop, pledge/unveil, W^X.
- **Supply chain**: cargo-vet, cargo-deny, SLSA Level 3 target, CycloneDX SBOM,
  reproducible builds, OSS-Fuzz integration.

---

## Known Limitations

- **SLSA provenance**: hash binding is a stub (ENG-080).  Provenance
  attestations are generated but hash binding is incomplete.
- **cargo install**: `--check-config` flag not yet implemented (ENG-085).
- **riscv64**: uses glibc (not musl) until the riscv64-musl target stabilises.
- **macOS**: development target only; sandbox profile not enforced automatically.

## Artefact Verification

All release binaries are signed with cosign (keyless OIDC, Sigstore transparency
log).  Verify a downloaded binary:

```sh
cosign verify-blob \
  --bundle heimdall-v1.0.0-x86_64-unknown-linux-musl.cosign.bundle \
  heimdall-v1.0.0-x86_64-unknown-linux-musl
```

SLSA Level 3 build provenance (`multiple.intoto.jsonl`) is attached.  Verify:

```sh
slsa-verifier verify-artifact \
  heimdall-v1.0.0-x86_64-unknown-linux-musl \
  --provenance-path multiple.intoto.jsonl \
  --source-uri github.com/FlavioCFOliveira/Heimdall \
  --source-tag v1.0.0
```

A signed CycloneDX SBOM (`sbom-v1.0.0.cdx.json`) is attached per THREAT-014.

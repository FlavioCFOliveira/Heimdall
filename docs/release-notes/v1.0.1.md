# Heimdall v1.0.1 — Release Notes

**Release date:** 2026-05-05
**Type:** LTS patch release (v1.0 branch)
**Tag:** `v1.0.1`

---

## Important: v1.0.0 was a non-functional placeholder

v1.0.0 shipped with a stub `fn main() {}` binary entry-point.  All library
crates were real and tested, but the server could not be started.

**v1.0.1 backports the binary wiring from Sprint 46 and is the first
operational build on the v1.0 LTS branch.**

---

## Changes

This release cherry-picks the Sprint 46 binary-wiring commits (tasks
#454–#465, #537–#540, #552–#556, #569) from the main development branch
onto the v1.0 LTS branch.

### Binary entry-point

- `heimdall start --config <path>` — fully operational boot sequence.
- `heimdall check-config --config <path>` — deep validation without starting.
- `heimdall version` — build metadata (version, git hash, build date, target).

### Boot sequence (BIN-015..BIN-019)

1. CLI parsing (clap, exit 64 on usage error).
2. Config load and validation.
3. Logging initialisation (JSON or pretty, configurable).
4. Tokio runtime start (io_uring on Linux, epoll/kqueue otherwise).
5. Admission telemetry initialisation.
6. Running-state construction (ArcSwap).
7. Signal handlers: SIGTERM → graceful drain, SIGINT → immediate exit,
   SIGHUP → hot config reload.
8. Role assembly: auth / recursive / forwarder from config.
9. Transport listener binding: UDP / TCP / DoT / DoH / DoQ.
10. Privilege drop (CAP_NET_BIND_SERVICE + unprivileged user).
11. sd_notify(READY=1); watchdog keepalive every WATCHDOG_USEC/2.
12. Admin-RPC UDS listener.
13. HTTP observability endpoint (/healthz, /readyz, /metrics, /version).
14. Redis pool bootstrap.
15. Drain coordinator with grace timeout from config.

### Other additions

- OS resource limits (NOFILE, NPROC, AS) at boot phase (THREAT-068).
- mimalloc as the default global allocator (ADR-0062).
- Canonical example config: `contrib/heimdall.toml.schema.json` (taplo schema).
- Graceful Redis pool drain on SIGTERM/drain-command.

---

## Upgrade from v1.0.0

v1.0.0 was non-functional.  Replace the binary; no configuration changes
are required.  All TOML schema and API surfaces are unchanged.

---

## LTS support policy

The v1.0 branch receives critical security fixes and the binary-wiring
backport for 12 months from the v1.0.0 tag (until 2027-04-27).  See
`docs/adr/0059-lts-model.md` for the full LTS policy.

## GitHub Discussions post

**Title:** Heimdall v1.1.0 released — first functional build

We are happy to announce Heimdall v1.1.0, the first functional General
Availability release of the Heimdall DNS server.

### What was wrong with v1.0.0?

v1.0.0 (tagged 2026-04-27) shipped with a binary entry-point that contained
only `fn main() {}`. The library crates (`heimdall-core`, `heimdall-runtime`,
`heimdall-roles`) were fully implemented and tested, but the binary that
operators would actually run was not wired up at that point.

We apologise for the confusion this caused.  The v1.0.0 tag was an
infrastructure release intended to validate the supply chain, OCI build
pipeline, and crates.io publication workflow — not a deployable server.

### v1.1.0 is deployable

The binary is now fully wired:

```
heimdall start --config /etc/heimdall/heimdall.toml
```

See the [quickstart](../../README.md) and the
[full release notes](v1.1.0.md) for details.

### LTS users: v1.0.1

The v1.0 LTS branch has also been updated.  v1.0.1 (tagged today) backports
the binary wiring onto the LTS branch.  If you are pinned to v1.0.x, upgrade
to v1.0.1.

## oss-security notification

Subject: Heimdall DNS server — v1.0.0 was non-functional; v1.1.0 is the
first functional release (not a security issue)

Hello,

This is an informational notice, not a vulnerability disclosure.

Heimdall (https://github.com/FlavioCFOliveira/Heimdall) is a high-performance
DNS server written in Rust.

v1.0.0 (2026-04-27) shipped with a binary entry-point of `fn main() {}`.
The server could not be started.  No security properties were claimed for
v1.0.0 because no code was executing.

v1.1.0 (2026-05-05) is the first operational release.  The binary is now
fully wired, implements the five-stage admission pipeline documented in the
threat model, and has passed the Sprint 53 stability and soak test suite.

v1.0.1 is simultaneously released as the first operational build on the
v1.0 LTS branch.

No CVE is requested.  We are notifying oss-security as a courtesy so that
security researchers who may have evaluated v1.0.0 are aware that the actual
server code is now live and merits fresh review.

Security contact: security@heimdall-dns.example (see SECURITY.md for PGP key).

## Minimum Supported Rust Version (MSRV)

**Rust 1.94.0** — required for Rust Edition 2024 features and `let`-chain stabilisation
(RFC 2497).  The workspace is pinned to the nightly channel via `rust-toolchain.toml`
until Rust 1.94 is released as stable.

## Key Features Delivered

- **DNSSEC validation**: Full RFC 4033–4035, aggressive NSEC/NSEC3 (RFC 8198), NSEC3
  iteration cap per RFC 9276, KeyTrap mitigations (RFC 9364), RFC 5011 trust anchor
  management, Negative Trust Anchor (NTA) store per RFC 7646.
- **QNAME minimisation**: Strict and relaxed modes per RFC 9156.
- **0x20 case randomisation**: Anti-spoofing per draft-vixie-dnsext-dns0x20.
- **Response Policy Zones (RPZ)**: QNAME/CIDR/NSDNAME tries, multi-zone first-match-
  wins, AD suppression + EDE, dynamic reload.
- **Admin RPC**: JSON over Unix domain socket — zone management, NTA, TEK rotation,
  rate-limit tuning, RPZ upsert, stats, drain, diag.
- **HTTP observability**: `/healthz`, `/readyz`, `/metrics` (Prometheus), `/version`.
- **Runtime hardening**: seccomp-bpf allow-list (THREAT-024), privilege drop to
  `heimdall` user + CAP_NET_BIND_SERVICE (THREAT-022/023), W^X linker flags
  (THREAT-027), OpenBSD pledge/unveil (THREAT-029).
- **Supply chain**: cargo-vet peer imports (Mozilla, Bytecode Alliance, Embark, Google),
  cargo-deny licence/source control, SLSA Level 3 provenance target (ADR-0058),
  CycloneDX SBOM (ADR-0056), reproducible builds (ADR-0057).

## Known Limitations (Pre-Audit)

- **External security audit**: not yet complete (Sprint 41 — in progress).  Do not
  deploy in hostile network environments until the Sprint 41 audit sign-off is published.
- **SLSA provenance**: the `slsa-github-generator` hash wiring is a stub (ENG-080).
  Provenance attestations are generated but hash binding is incomplete.
- **cargo install**: the binary lacks a stable configuration schema; the
  `--check-config` flag is not yet implemented.
- **riscv64**: uses glibc (not musl) until the riscv64-musl target stabilises.
- **macOS**: development target only; sandbox profile (contrib/macos/heimdall.sb)
  is not enforced automatically.

## Artefact Verification

All release binaries are signed with cosign (keyless OIDC, Sigstore transparency log).
Verify a downloaded binary:

```sh
cosign verify-blob \
  --bundle heimdall-v0.9.0-alpha.1-x86_64-unknown-linux-musl.cosign.bundle \
  heimdall-v0.9.0-alpha.1-x86_64-unknown-linux-musl
```

A signed CycloneDX SBOM (`sbom-v0.9.0-alpha.1.cdx.json`) is attached to this release.
